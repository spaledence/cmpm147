Experiment 03 - Alternative Worlds




Description

Generate an overworld and dungeon landscape with some autotiling and editable with the textboxes underneath.
Overworld Codes
"_" = grass
"w" = water
"." = edge
"h" = house
"^" = dirt
Dungeon Codes
"_" = wall
"." = room floor
"c" = chest
"p" = path
"^" = dirt
Micro Tileset - Overworld & Dungeon by thkaspar

Technical

First the textbox is created and filled in with the constraints that we set based on how we want our map to look. From there, we place tiles in said locations of the grid based on the value of the variable in each grid row, column pair. These tiles are customizable and ultimately make up the visual aspect of our world.

Reflection

Honestly, this project was a lot more work than I initially expected. I went into this project starting a late due to another class project, and I was not aware of how much work it actually was. It took me a while to figure out how exactly the tiles were working and it was especially hard for me to understand how to implement auto tiling given the instructions we had. I actually read through the tutorial link and I feel like I have a decent understanding of how auto tiling works, but I was not able to make it work properly in my program. I did give it several attempts though and got a pseudo method of it working, but my tile offsets for corners and sides were not how I wanted them, and I feel like at this point I just need to turn something in that works, so I scrapped what I had.
